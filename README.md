# Wedding RSVP Landing Page 💍

Этот проект представляет собой премиальный одностраничный лендинг для свадебного приглашения с поддержкой двух языков (KZ/RU), таймером и интеграцией с Google Таблицами.

## Структура проекта
- `index.html` — Главная страница.
- `style.css` — Стили и анимации.
- `script.js` — Логика (таймер, музыка, форма).
- `Code.gs` — Код для Google Apps Script (Бэкенд).
- `music.mp3` — Ваша фоновая музыка.
- `wedding_photo.jpg` — Главное фото молодоженов в арке.

## Пошаговая инструкция по деплою

### 1. Настройка Google Таблиц (Бэкенд)
1. Создайте новую **Google Таблицу**.
2. В верхнем меню выберите: **Extensions (Расширения)** -> **Apps Script**.
3. Удалите весь стандартный код и вставьте содержимое файла `Code.gs`.
4. Нажмите кнопку **Save (Сохранить)** (иконка дискеты) и назовите проект, например, "Wedding RSVP".
5. Нажмите кнопку **Deploy (Развернуть)** -> **New Deployment**.
6. Выберите тип: **Web App**.
7. В поле "Who has access" выберите: **Anyone** (это критично для работы формы).
8. Нажмите **Deploy**.
9. **ВАЖНО:** Скопируйте полученный **Web App URL**.

### 2. Подключение формы к сайту
1. Откройте файл `script.js`.
2. Найдите строку: `const GOOGLE_SCRIPT_URL = 'ВАШ_URL_ЗДЕСЬ';` (примерно 150-я строка).
3. Вставьте туда ваш **Web App URL**, скопированный на предыдущем этапе.
4. Сохраните файл.

### 3. Загрузка на GitHub
1. Создайте новый репозиторий на GitHub.
2. Загрузите туда файлы: `index.html`, `style.css`, `script.js`, `music.mp3` и ваше фото (переименуйте его в `wedding_photo.jpg` или измените имя в HTML).
3. Перейдите в **Settings** (репозитория) -> **Pages**.
4. В разделе "Build and deployment" выберите ветку `main` и папку `/ (root)`. Нажмите **Save**.
5. Через пару минут ваш сайт будет доступен по ссылке вида `https://ваш-логин.github.io/имя-репозитория/`.

### 4. Исправление проблемы с фото
Если вы не видите фото в арке:
- Убедитесь, что файл с фотографией лежит в той же папке, что и `index.html`.
- Убедитесь, что название файла в папке **точно совпадает** с названием в `index.html` (включая расширение .jpg или .png).
- Рекомендуется использовать вертикальное фото высокого качества.
